Connection Flow
User → Nginx → Backend/Frontend (HTTPS)
Backend ↔ PostgreSQL · MinIO (R/W)
Backend → Redis → Celery Worker (Queue)
Backend / Worker → Gateway → External AI (Egress)
Backend → Frontend (SSE Streaming)
1
📤 Upload Document
ผู้ใช้สร้าง Job แล้วอัปโหลดไฟล์ PDF หรือรูปภาพ
ไฟล์ถูกบันทึกลง MinIO (Object Storage)
สร้าง Document record ใน DB — status: uploaded
Frontend Upload POST /documents/upload MinIO Storage PostgreSQL
2
🔍 OCR Processing
Backend enqueue Celery Task → status: ocr_processing
Worker ดาวน์โหลดไฟล์จาก MinIO ส่งไปยัง OCR Engine ทีละหน้า
บันทึก ocr_text, ocr_pages, page_count → status: ocr_completed
Celery Worker Redis Queue External OCR API Per-page Processing
3
🧠 Structure Extraction (AI)
นำ OCR text + JSON Schema ที่กำหนดไว้ส่งให้ LLM
LLM แปลงข้อความ → Structured JSON ตาม Schema
Parse/Clean response (รองรับ markdown code block, wrapper fields)
บันทึก extracted_data → status: extraction_completed
LLM API JSON Schema Structure Extract Endpoint
4
✅ Human Review
ผู้ใช้ตรวจสอบ extracted_data บน UI เปรียบเทียบกับเอกสารต้นฉบับ
แก้ไขข้อมูลได้โดยตรง → บันทึกเป็น reviewed_data
Approve → status: reviewed
Reject → ส่งกลับไป Reprocess ใหม่ได้
Manual Review UI POST /decision Re-process if Rejected
5
💬 ChatDOC — AI Q&A
ผู้ใช้เปิด Chat panel ภายใน Job เดียวกัน
Backend รวม context จากทุก Document (OCR text + reviewed_data)
ส่งพร้อม prompt ไปยัง LLM Integration ที่เลือก → Stream SSE กลับ
บันทึก conversation history (ChatConversation / ChatMessage)
SSE Streaming Multi-doc Context Conversation History Thai / English
6
🔗 Integration & Output
LLM Integration — ส่ง document context ให้ LLM วิเคราะห์เพิ่มเติม (streaming)
API Integration — Push ข้อมูลออกไปยัง Webhook / External API
Workflow Integration — Trigger automation pipeline
LLM · API · Workflow Streaming SSE Result Stored
7
🌐 External API Access
ระบบภายนอก เรียก InsightDOC ผ่าน API Token โดยตรง
สร้าง Job · Upload Document · Trigger Process · ดึงผลลัพธ์
รองรับ Integration กับ RPA / ERP / ระบบ Enterprise อื่นๆ
API Token Auth POST /external/jobs GET /external/documents
🔗 Integration Types
🤖 LLM Integration
เชื่อมต่อ LLM Provider แบบ OpenAI-compatible

Providers: GPT-4o · Claude · Gemini · Ollama · DeepSeek · ฯลฯ

ใช้สำหรับ:
· วิเคราะห์เอกสาร (Send LLM)
· ChatDOC (Q&A)
· Streaming response (SSE)
· Reasoning model support

· บันทึก Integration Result ลง DB
· รองรับ custom headers & auth

📊 Data Flow Summary
Document Status Flow
📥 uploaded ↓ Celery Task ⚙️ ocr_processing ↓ OCR API ✅ ocr_completed ↓ LLM Extract 🧠 extraction_completed ↓ Human Review ✅ reviewed
Security Design
🛡️ Nginx SSL termination
🔒 Internal network isolation
🛡️ Gateway = egress-only proxy
🔑 JWT + API Token auth
🚫 MinIO ไม่ expose ออกนอก
🚫 DB/Redis ไม่ expose ออกนอก
Memory Limits
🔀 Nginx: 256 MB
🐍 Backend: 1 GB
⚡ Celery Worker: 2 GB
🖥️ Frontend: 512 MB
🐘 PostgreSQL: 1 GB
🔴 Redis: 256 MB
🗄️ MinIO: 1 GB
